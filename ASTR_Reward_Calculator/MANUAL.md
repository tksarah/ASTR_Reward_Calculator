# ASTR Reward Tax Calculator 利用ガイド

## はじめに

このアプリは、Astar Network の dApp Staking で受け取ったリワード情報を、ブラウザ上で記録、集計、保存するためのツールです。

できることは主に次のとおりです。

- Claim 情報の記録
- USD と JPY の自動計算
- 月別リワードの確認
- CSV、JSON、PDF での出力
- JSON バックアップの読み込み

このアプリはブラウザだけで動作し、入力したデータはブラウザの LocalStorage に保存されます。

## 最初に知っておきたいこと

- 参考レート取得機能は補助機能です
- 取得値は正確な数値を保証するものではありません
- 外部サービスの状況や日付によっては、レートを取得できない場合があります
- 重要なデータは JSON エクスポートで定期的にバックアップしてください
- 税務判断の前に、必ず税理士または税務署へ確認してください

ASTR レートを自分で確認したい場合は、次のサイトを利用できます。

- CoinGecko: https://www.coingecko.com/ja

## 使い始める

### 起動方法

Python を使う場合は、次のコマンドを実行してください。

```bash
cd c:\Users\sarah\Documents\myapp\ASTR_Reward_Calculator
python -m http.server 8000
```

ブラウザで次の URL を開きます。

```text
http://localhost:8000
```

Node.js の http-server を使う場合は、次のコマンドでも起動できます。

```bash
cd c:\Users\sarah\Documents\myapp\ASTR_Reward_Calculator
http-server -p 8000
```

## 基本の使い方

このアプリは、次の流れで使うと分かりやすいです。

1. 日付を入力する
2. 必要なら参考レートを取得する
3. Claim 量、為替レート、ASTR レート、Fee を確認して入力する
4. 追加ボタンで保存する
5. 右側の集計結果と月別グラフを確認する
6. 必要に応じて CSV や JSON で保存する

## 画面の見方

### 左側: リワード情報を入力

1件ずつ Claim 情報を入力するフォームです。

入力する項目は次の 5 つです。

- 日付
- Claim 量（ASTR）
- 為替レート（JPY/USD）
- ASTR レート（USD/ASTR）
- トランザクション Fee（ASTR）

### 右上: 集計結果

入力済みデータの合計値をまとめて表示します。

確認できる主な項目は次のとおりです。

- Claim回数
- 合計ASTR量
- 合計Fee（ASTR / JPY）
- 合計リワード（USD / JPY）
- 加重平均取得ASTR単価（USD / JPY）
- 最終損益額（JPY）

### 右中央: 月別集計グラフ

月ごとの合計リワード額を棒グラフで確認できます。

### 右下: Claim 履歴

登録したデータの一覧です。ここから編集、削除、並び替えができます。

## リワード情報を入力する

### 1. 日付を選ぶ

まず、Claim 日付を入力します。

### 2. 参考レートを取得する

必要に応じて「その日の参考レートを取得」を押してください。

この操作で、次の参考値が自動入力されます。

- ASTR/USD
- USD/JPY

ただし、参考レートは補助情報です。取得できない場合や、期待した値と違う場合は、手動で入力または修正してください。

### 3. 数値を確認して入力する

次の項目を確認して入力します。

- Claim 量
- 為替レート
- ASTR レート
- Fee

参考レートを取得したあとでも、手入力で自由に上書きできます。

### 4. 追加する

「追加」を押すと、その 1 件が保存されます。

保存後は、次の表示が自動で更新されます。

- 集計結果
- 月別集計グラフ
- Claim 履歴

## あとから修正する

### 編集する

Claim 履歴の「編集」を押すと、対象データがフォームに戻ります。

内容を修正して「更新」を押すと、既存データに上書き保存されます。

### 削除する

Claim 履歴の「削除」を押すと、そのデータを削除できます。

## 表の見方と並び替え

Claim 履歴の表では、列見出しを押すと並び替えができます。

並び替えできる項目は次のとおりです。

- 日付
- ASTR量
- USD
- JPY

データが 6 件以上になると、ページ送りが表示されます。1 ページに 5 件ずつ表示されます。

## 集計結果の見方

このアプリでは、入力した各データから自動で金額計算を行います。

主な見方は次のとおりです。

- 合計リワード（USD / JPY）: 受け取ったリワードの合計額です
- 合計Fee（JPY）: 手数料を円換算した合計です
- 加重平均取得ASTR単価: 登録データ全体から見た平均単価です
- 最終損益額（JPY）: 総リワードから総Feeを差し引いた金額です

最終損益額（JPY）は次の考え方で計算されます。

```text
総リワード（JPY） - 総Fee（JPY）
```

## 月別グラフを見る

月別集計グラフでは、月ごとの合計リワード額（JPY）を確認できます。

次の操作を行うと、自動で再描画されます。

- データを追加したとき
- データを削除したとき
- JSON をインポートしたとき
- テーマを切り替えたとき

## データを保存する

このアプリのデータは、ブラウザの LocalStorage に保存されます。

保存される主な内容は次のとおりです。

- リワードデータ
- テーマ設定

次のような場合、データが消えることがあります。

- ブラウザのストレージを削除した場合
- サイトデータを消去した場合
- 別のブラウザや別の PC で開いた場合

大切なデータは、JSON エクスポートでこまめにバックアップしてください。

## エクスポートとインポート

### CSV エクスポート

一覧と集計結果を CSV ファイルとして保存します。

### JSON エクスポート

バックアップ用の JSON ファイルを保存します。

### PDF エクスポート

レポート内容を PDF ファイルとして直接保存します。

### レポート出力

ウォレットアドレス、ウォレット名、メモを任意入力したうえで、印刷用画面を開きます。

### JSON インポート

以前に保存した JSON ファイルを読み込みます。

### データ削除

確認ダイアログのあと、登録済みデータをすべて削除します。

## テーマを切り替える

画面右上のボタンから、ライトモードとダークモードを切り替えできます。

選択したテーマはブラウザに保存され、次回起動時にも引き継がれます。

## よくある注意点

### 参考レートが取得できない

外部サービスの状況や対象日付によっては取得できないことがあります。その場合は手動で入力してください。

### 金額が想定と違う

参考レートは補助機能です。必要に応じて、ご自身で確認したレートに修正してください。

### データが消えた

LocalStorage が削除されるとデータは復元できません。定期的に JSON エクスポートを行ってください。
